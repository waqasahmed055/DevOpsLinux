# Annual Goals 2026-2027 | Linux System Administrator

## Goal 1: Automation, Patching, and Vulnerability Remediation (Enhanced)

**Goal:**  
Continue and mature the automated patch management and vulnerability remediation workflow across all Linux servers, with improved version control and reliability.

**Objectives:**
- Maintain and enhance Ansible-based automation for patching and configuration management.
- Integrate all Ansible playbooks, roles, and custom scripts into **GitHub** for version control, collaboration, and rollback safety.
- Achieve and sustain **≥90% patch compliance** within 30 days of patch release across production systems.
- Remediate vulnerabilities detected by security scanning tools on a prioritized schedule.
- Document monthly patch status, vulnerability trends, and automation improvements in the centralized repository.

## Goal 2: Documentation and Environment Cleanup (Continued & Strengthened)

**Goal:**  
Sustain a clean, well-documented Linux environment to reduce technical debt and improve operational excellence.

**Objectives:**
- Maintain full inventory of systems, services, and configurations in the centralized documentation repository.
- Identify and decommission remaining obsolete systems, unused services, and legacy configurations.
- Ensure **100% of production systems** have up-to-date standardized documentation.
- Continuously improve documentation quality and accessibility (e.g., READMEs, runbooks, architecture diagrams in GitHub).

## Goal 3: Authentication Standardization (Completion & Maintenance)

**Goal:**  
Complete the migration to centralized authentication and maintain secure access controls.

**Objectives:**
- Finalize migration of remaining local accounts to LDAP/Active Directory.
- Run and automate the **PowerShell script** to map POSIX attributes for LDAP integration in AD.
- Implement and enforce group-based access controls across all Linux systems.
- Ensure full compliance with company security policies and conduct quarterly access reviews.

## Goal 4: Centralized Logging and Monitoring for Application Servers

**Goal:**  
Implement centralized log collection and monitoring for improved troubleshooting and visibility, especially for application servers (GAWS, DegreeWorks, etc.).

**Objectives:**
- Deploy **Grafana + Alloy** (or Fluentd/Loki stack) for log collection from all Linux and application servers.
- Configure structured logging with labels for quick filtering and searching.
- Set up centralized Grafana dashboards for real-time monitoring, alerting, and troubleshooting.
- Eliminate manual log file sharing by enabling team-wide access via Grafana.
- Establish retention policies and ensure compliance with security standards.

## Goal 5: Infrastructure as Code & Process Improvement

**Goal:**  
Strengthen Infrastructure as Code practices and operational efficiency.

**Objectives:**
- Store all infrastructure-related code (Ansible, scripts, Terraform if applicable) in GitHub with proper branching, reviews, and CI/CD where possible.
- Implement automated testing for critical playbooks.
- Conduct quarterly environment health audits and produce improvement reports.
- Explore and pilot additional automation opportunities (e.g., self-service tools, configuration drift detection).

---

**Success Metrics (Overall):**
- All playbooks/scripts in GitHub with meaningful commit history.
- Reduced mean time to resolution for incidents through better monitoring.
- Zero critical vulnerabilities older than 30 days.
- Fully documented and standardized Linux environment.

These goals build upon last year’s achievements while introducing modern practices (GitHub, centralized observability) to further increase efficiency and security.
